About

I'm an Application Security Engineer with a focus on web application security, penetration testing, secure code review, and threat modeling. My work sits at the intersection of attack and defense — understanding the conditions that make vulnerabilities possible, so teams can build systems that are genuinely harder to break.

I've developed the Vulnerability Conditions Framework — a methodology for decomposing web vulnerabilities into their required preconditions, making security analysis more systematic and less dependent on pattern-matching.

Beyond technical work, I speak at security conferences, write about application security concepts, and contribute to making security knowledge more accessible to engineers and practitioners.
